# Hermes Training Plan

## 1. 最終目標

Hermesは、Codexを使った開発に必要な基礎知識を前提として、Hermes固有の追加知識 alpha を最小化しながら、小規模から中規模の開発課題を自律的に遂行できることを目指す。

人間は原則として、以下のみを行う。

- 課題提示
- 方針確認
- 最終レビュー

## 2. 対象領域

優先対象は以下とする。

- Python
- Colab教材
- 小規模CLIツール
- pytest付きリポジトリ
- Keras / PyTorch / データ分析コード
- 授業・研究用の補助ツール

## 3. alphaを小さくする方針

Hermes固有の追加概念は、原則として以下に限定する。

- Task Pack
- Skill
- Run Log
- Eval
- Intervention

## 4. Hermesの標準行動

Hermesは課題を受け取ったら、原則として次を行う。

1. README / 設定ファイル / tests を確認する
2. 実行方法とテスト方法を推定する
3. 失敗を再現する
4. 最小修正を行う
5. 関連テストを実行する
6. 差分と確認結果を報告する
7. 人間に質問する前に、自分で確認できることを確認する

## 5. Codexの役割

CodexはHermesの育成係として、以下を行う。

- 日次訓練課題の作成
- Hermes dashboardへの課題投入
- Hermes出力の保存
- 実装結果のテスト
- 評価
- 失敗分析
- スキル追加案の作成
- 翌日の訓練計画の更新

ただし、スキルファイルや評価基準の変更は原則としてPRで提案し、人間レビューを受ける。

## 6. 評価指標

- 完遂率
- テスト通過率
- 人間介入回数
- 人間入力文字数
- Hermesからの質問回数
- 修正差分の小ささ
- 同じ失敗の再発率
- 既存Skillの再利用率

## 7. Codexへの制約

CodexはHermesを過保護にしすぎない。Hermesが自分で解ける部分は自分で解かせる。

Codexは、Hermesの失敗を単に修正するのではなく、失敗をSkillまたはFailure Patternに変換する。
